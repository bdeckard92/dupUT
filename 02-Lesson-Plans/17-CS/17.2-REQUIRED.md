* Open `03-Factory-Function/Solved/index.js` in your IDE and explain the following: 

    * First we need to create three variables for the `type`, `price`, and `origin` of the beverage. You can assign them to values of your choice.

    ```js

    const price = 6;
    const origin = 'Kenya';
    ```

    * üîë Next, we need to return an object literal which contains the methods `product()`, `amount()`, and `place()`. Each method will print out a message about the beverage. Notice how they have access to the variables we created above!

    ```js
    return {
        product: function() {
        console.log(`Would you like a ${type}?`);

        amount: function() {

        }
    };
    ```

    * üîë Finally, we need to invoke the methods `product()`, `amount()`, and `place()`.

    ```js
    drink.place();
    ```

* Ask the class the following questions (‚òùÔ∏è) and call on students for the answers (üôã):

    * ‚òùÔ∏è In what ways are using factory functions better than using classes?

    * üôã Both behave similarly and can create objects but use different patterns. The OOP pattern needs to use keywords like `new` and `this` which aren't intuitive. Always needing to remember to add the `new` keyword for object creation can also lead to errors. Different errors with `this` due to context changes can lead to bugs. The factory function, on the other hand, has no need of these constructs and looks much more like straight forward functions we are used to in JavaScript.

    * ‚òùÔ∏è What can you do if you don't completely understand this?

* Open `04-Fibonacci/Solved/fibonacci.js` in your IDE and explain the following: 

    * We are only going to use the linear approach for this exercise. But if you want to see the recursive approach, it will be included in the solution code.

    * üîë First, we create an array with the first two numbers of the sequence.

    ```js
    let arr = [1, 1];
    ```

    * üîë The next number in the array will be at index `2` so we need to loop from `2` to the number requested. We can do that with a `for` loop. We will add to the array the sum of the previous two numbers in the array.

    ```js
    for (let i = 2; i <= num; i++) {
        arr.push(arr[i - 2] + arr[i - 1]);
    }
    ```

    * üîë Finally, when we reach the point in the array of the given number, we stop the loop and return that number, which is the last number in the array.

    ```js
    return arr[num - 1];
    ```

### 11. Instructor Review: Regex  (10 min) 

* Bring students back from breakout rooms. 

* Use the prompts and talking points below to review the following key (üîë) points:

    * ‚úîÔ∏è `^` and `$`

    * ‚úîÔ∏è `[]`

    * ‚úîÔ∏è `{}`

* Open `05-Regex/Solved/regex.md` in your IDE and explain the following: 

    * There are many components involved in this expression. Here is the breakdown of each of those components.

    * üîë First, we use `^` to match the beginning of a string and `$` to match the end of the string

    ```regex
    /^   $/
    ```

    * üîë Then we use the `[]` to enclose the types of characters we are matching. In our case, that's any lowercase letter between `a-z`, any number between `0-9`, underscore `_`, and a hyphen `-`.

    ```regex
    [a-z0-9_-]
    ```

    * üîë Finally we use a quantifier `{}` to specify that we want it to be between 3-16 characters long.

    ```regex
    {3, 16}
    ```
