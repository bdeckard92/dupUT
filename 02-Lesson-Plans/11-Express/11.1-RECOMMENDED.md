    * âœ”ï¸ `app.get('/api/characters/:character')`

* Open `02-Stu_Parameters/Solved/server.js` in your IDE and explain the following: 

    * ðŸ”‘ We set up the route to handle variable content in the URL so we can pass in different character names. This is called a "route parameter".

    ```js
    // i.e. /api/characters/obiwankenobi
    app.get('/api/characters/:character', (req, res) => {});
    ```

    * ðŸ”‘ We can figure out which character we are looking for by accessing the route's parameter in the param's object and look for a matching data in our array of characters to send back to the client as JSON.

    ```js
    const chosen = req.params.character;

    for (let i = 0; i < characters.length; i++) {
        if (chosen === characters[i].routeName) {
            return res.json(characters[i]);
        }
    }
    ```

    * If we don't find a character, we send back a message saying so because no matter what, we have to respond to a request.

* Ask the class the following questions (â˜ï¸) and call on students for the answers (ðŸ™‹):
