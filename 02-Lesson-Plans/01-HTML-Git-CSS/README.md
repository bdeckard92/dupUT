## Overview

In this module, students learn three web development technologies that are essential to their success as a web developer. Students learn HTML as they create the basic structure for webpages. They learn CSS so that they can organize and style elements on the page. Finally, they learn how ot use Git and navigate the file system using the command line.

In this week's virtual classes, you will welcome students to the first module of the course. You'll also teach them about the basics of HTML, attributes, CSS selectors, the CSS Box Model, and CSS positioning.

## Overview of Asynchronous Material

This week in the asynchronous content, students are introduced to HTML, CSS, and Git. These skills are important for the following reasons:

* HTML, CSS, and Git are the three fundamental technologies of web development.

* HTML allows students to scaffold webpages so they can display content to the end user. A solid foundation in HTML will help students learn more complex skills later in the course.

* CSS is an important skill to master, as it allows students to style the content on a webpage and organize their code so that style is separate from content.

* Git is an essential skill for developers, and employers will expect new hires to have a firm grasp of it on day one of any new position. Being well-practiced in essential Git commands will allow students to effectively save their work and collaborate effectively.

Throughout the async module this week, students will use HTML, CSS, and Git to do the following:

* Identify the difference between HTML and CSS, what each technology does, and how they interact with each other.

* Learn semantic HTML and the best practices for structuring HTML files.

* Structure HTML elements to satisfy specific accessibility standards that allow tools such as screen readers to explain a webpage's content in a meaningful way to people with disabilities.

* Follow best practices for structuring CSS files.

* Add comments to CSS files to make the code easier to read.

* Publish a webpage to the internet using GitHub Pages.

## Module Challenge

For the [Module 01 Challenge](../../01-Class-Content/01-HTML-Git-CSS/02-Challenge), students will refactor an existing webpage that currently falls short of acceptable accessibility standards for web development. Students will make the page easier to access by implementing a semantic structure of HTML elements and adding `alt` attributes to images. These changes will improve SEO and allow accessibility tools, such as screen readers, to effectively parse and communicate the page's layout and contents.

Students must know how to implement the following skills in order to succeed in this week's Challenge:

* Using the DRY design pattern and consolidating CSS selectors

* Ensuring that all links function properly

* Implementing semantic structure of HTML elements

* Writing well-documented code using comments

* Implementing the `alt` attribute for images

* Ordering elements in sequence
